# 游戏退出与脱敏诊断

> 启动器源码版本：`0.9.1`
>
> 状态：本地退出记录、异常退出提示和玩家主动生成诊断包已完成；网络上传尚未实现

## 1. 目标

诊断能力用于回答三个问题：

1. Minecraft 是正常退出、异常退出，还是启动器没有取得退出码。
2. 当前客户端的最新日志和崩溃报告是否包含可定位的问题。
3. 玩家能否在不暴露账号凭据、世界存档和本机身份的前提下，把必要信息交给管理员。

启动器不会因为诊断功能改变 Minecraft 服务端、Velocity 或任何游戏进程的启动策略。

## 2. 本地文件

退出记录和诊断包位于：

```text
%LocalAppData%\Hechao\Launcher\
|-- game-exits.json
`-- diagnostics\
    `-- Hechao-Diagnostic-<UTC 时间>-<随机 ID>.zip
```

- `game-exits.json` 最多保留 20 条，只记录档案 ID、Minecraft 进程 ID、退出码和起止时间。
- `diagnostics` 最多保留 10 个由玩家主动生成的 ZIP。
- 卸载启动器不会自动删除这些本地支持记录。
- 诊断包不会写入游戏数据根目录，也不会进入客户端发布清单。

## 3. 退出记录

Minecraft 进程成功启动后，启动器订阅进程退出事件：

- 退出码 `0`：记录为正常退出。
- 非零退出码：记录为异常退出，并在启动器仍运行时显示提示。
- 无法取得退出码：记录为状态未知。

“游戏启动后关闭启动器”启用时，启动器退出后无法继续接收进程事件，因此本次退出可能没有记录。玩家下次打开启动器后仍可从设置页主动生成诊断包，因为 Minecraft 自己写入的日志和崩溃报告仍在档案目录中。

退出事件订阅者的异常不会阻止游戏进程对象清理，也不会影响下一次启动。

## 4. 诊断包内容

玩家在设置页选择“生成诊断包”后，启动器只读取当前服务器对应档案中的：

- `logs\latest.log`；
- `crash-reports` 目录中最新的 `crash-*.txt`；
- 启动器版本、档案 ID、已安装档案版本、Windows/.NET 架构；
- 与当前档案匹配的最近一次退出码和时间。

每个日志文件最多读取末尾 512 KiB。超过限制时丢弃较早内容并写入截断标记，避免异常日志耗尽内存或生成超大压缩包。

固定不会收集：

- `saves` 世界存档；
- 截图、资源包、光影包和模组配置内容；
- 赫朝账号密码；
- Microsoft、Xbox 或 Minecraft 会话缓存；
- 启动器刷新令牌；
- 浏览器历史或其他应用日志。

## 5. 脱敏规则

写入 ZIP 前会替换：

- `Bearer`、Xbox、OAuth、访问令牌和刷新令牌；
- JWT 形态的令牌；
- UUID、长数字账号标识和邮箱；
- IPv4 地址；
- 当前赫朝账号、Minecraft 玩家名和账号标识；
- Windows 用户名、计算机名、用户目录；
- 当前赫朝游戏数据根目录。

固定 ZIP 路径由启动器生成，不采用日志内容作为 ZIP 条目名。日志目录、崩溃目录和源文件若是重解析点会被拒绝。

诊断包仍可能包含玩家在游戏聊天或模组日志中主动输入的其他自由文本。对外发送前，玩家和管理员仍应按最小披露原则处理。

## 6. 玩家操作

1. 打开“设置”。
2. 在“故障诊断”查看最近一次游戏退出状态。
3. 选中发生问题的服务器档案。
4. 点击“生成诊断包”。
5. 启动器完成脱敏后打开本地诊断目录。

“打开诊断目录”只打开本地文件夹，不上传文件。

## 7. 管理员处理

收到诊断包后：

1. 先读取 `diagnostic.json`，确认启动器版本、档案 ID、档案版本和退出码。
2. 查看 `crash-reports`；存在明确崩溃报告时优先处理其异常栈。
3. 再查看 `logs/latest.log` 末尾，定位加载器、模组、Java、显卡驱动或网络错误。
4. 不要求玩家补交 `saves`、账号缓存或完整用户目录。
5. 问题解决后删除管理员侧临时副本。

## 8. 验证基线

`0.9.1` 自动测试覆盖：

- 最新退出记录和 20 条保留上限；
- 日志、崩溃报告和元数据进入固定 ZIP 条目；
- 世界存档不会进入 ZIP；
- 账号名、路径、令牌、UUID、邮箱和 IP 脱敏；
- 超大日志只保留末尾 512 KiB；
- 只选取最新崩溃报告；
- 本地诊断包最多保留 10 个。

正式发布前还要在真实 Minecraft 正常退出和一次受控崩溃场景中各验证一次。

## 9. 尚未实现

当前没有诊断上传 API，也不会在后台自动发送日志。

未来若增加“上传给管理员”，必须同时完成：

- 玩家在动作发生前明确确认；
- 使用赫朝账号鉴权和短时上传授权；
- 服务端文件大小、格式、频率和保存期限限制；
- 管理后台下载权限与审计；
- 自动到期删除；
- 更新隐私说明和玩家安装手册。
